# Codex Dream Skin · Windows

## 自定义安装目录修复

- 修复把程序安装到 D 盘或其他自定义目录后，安装结束时可能出现 `CreateProcess failed; code 267` 的问题。
- 程序安装目录与主题状态目录现在是两个明确边界；安装、启动、更新、恢复和卸载始终使用同一个显式状态目录。
- 安装结束后的托盘任务与所有快捷方式使用用户选择的程序目录作为工作目录，不再假设程序安装在默认位置。
- 如果 1.4.1 安装失败，直接运行本次 `Setup.exe` 覆盖安装即可，不需要手动清理；当前主题、已保存主题和导入图片默认保留。

## 已安装旧版本的用户

- 已成功安装 1.4.1 的用户可以通过托盘“检查更新”升级。
- `windows-v1.3.3` 或 `windows-v1.4.0` ZIP 用户需要下载并运行一次本次 `Setup.exe`；不需要先卸载或恢复官方主题。
- 更新清单与安装包都由 GitHub Release 托管，安装包必须通过清单中的 SHA-256 校验后才会启动。

## 全新安装

下载 `Codex-Dream-Skin-Windows-*-Setup.exe` 后双击运行。安装器按当前用户安装，不要求管理员权限。

## 卸载

在 Windows“已安装的应用”或开始菜单中卸载 `Codex Dream Skin`。卸载时可以选择保留主题数据，方便以后重新安装，也可以一并清除。

## 安全声明

非 OpenAI 官方产品。主题通过本机回环 CDP 加载，不修改 WindowsApps、`app.asar` 或官方代码签名。未签名构建可能触发 Windows SmartScreen；正式分发建议配置代码签名证书。
